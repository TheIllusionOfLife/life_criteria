## レビュー指摘への対応方針

### C1. 理論的基盤 → Phase 0で対処

教科書的7基準を出発点としつつ、以下を文書化する:

1. **基準体系の比較表**: 7基準 vs. NASA定義 vs. オートポイエーシス vs. Ruiz-Mirazo条件
2. **7基準を採用する理由**: 「最も広く認知された基準であり、各基準が独立した計算プロセスとして実装可能」
3. **Weak ALifeとしてのframing**: 「生命そのもの」ではなく「生命の機能的性質を再現するモデル」として位置づける
4. **オートポイエーシスとの接続**: active boundary + metabolism = 計算的オートポイエーシス

**担当**: ドメイン専門家と協議の上、論文のIntroduction/Related Workセクションに組み込む

### C2. 評価指標 → Phase 1のパイロット実験で校正

閾値を先験的に決めるのではなく、以下のプロセスで決定する:

1. Phase 1でベースライン実験を実施
2. ランダムベースラインとの統計的有意差をもって「生命的」と判定
3. 具体的な統計手法: Mann-Whitney U検定 or permutation test
4. N値・摂動強度は予備実験の分布から決定し、論文にはその決定過程を記述

### H1. 締切修正 → 即時

**2026-04-01** を正式な締切とする。本計画のスケジュールはこれに基づく。

### H2. 比較設計 → Phase 0 + Phase 4

1. Phase 0で比較対象システムの7基準充足度マトリクスを作成
2. 定量比較は「同じメトリクスを走らせる」のではなく、**文献報告値との比較 + 質的比較**とする
3. 論文のTableとして: 各システムの基準ごとの実装深度を5段階で評価

### H3. 統合リスク → ピボット戦略を策定

統合リスクを **4.0/5** に上方修正。以下のピボット戦略を設定:

| ハイブリッド二層構造が不安定 | Swarmを落とし、Agent-basedに単純化 |

## 8週間スケジュール（2026-02-08 → 2026-04-01）

### Week 1 (02/08-02/14): Phase 0 — 理論基盤 + 環境構築

**目標**: 理論的正当化の草稿 + 開発環境の準備

- [ ] 基準体系比較表の作成（7基準 vs. NASA vs. オートポイエーシス vs. Ruiz-Mirazo）
- [ ] 既存システム比較表の作成（Polyworld, ALIEN, Flow-Lenia, Avida, Lenia, Coralai）
- [ ] オートポイエーシスとの理論的接続の草稿
- [ ] Rust + Python プロジェクトセットアップ（PyO3 or maturin）
- [ ] 連続2D環境の基本フレームワーク実装
- [ ] ローカルLLM (Ollama) のベンチマーク: 推論速度 × エージェント数のスケーラビリティ測定

**Go/No-Go (Week 1末)**:
- ローカルLLMが50エージェントの同時推論を実用的速度で処理できるか？
- No → LLMを高次判断のみに限定し、低レベル行動は進化的NNに切り替え

### Week 2-3 (02/15-02/28): Phase 1A — 代謝 + 細胞組織化

**目標**: 単一organismが自律的に維持される最小システム

**代謝**:
- [ ] リソースタイプの決定（予備実験で3種類を比較テスト）
  - 候補A: 抽象エネルギートークン（シンプル、制御しやすい）
  - 候補B: 空間的リソース（光、栄養素の空間分布）
  - 候補C: 情報リソース（環境からの情報を処理して有用な形に変換）
- [ ] グラフベース代謝ネットワーク実装（Rustコア）
- [ ] 代謝ネットワークの遺伝的エンコーディング
- [ ] 廃棄物排出メカニズム

**細胞組織化**:
- [ ] swarmエージェントの基本実装（位置、速度、状態）
- [ ] アクティブ境界維持プロセス（代謝コスト消費）
- [ ] 境界崩壊メカニズム（維持コストが払えないと溶解）

**Go/No-Go (Week 3末)**:
- 単一organismが1000タイムステップ自律維持できるか？
- No → 代謝をODEベースに切り替え（ピボットA）

### Week 4 (03/01-03/07): Phase 1B — 恒常性 + 刺激応答

**目標**: 環境摂動からの回復能力

- [ ] 内部状態ベクトルの実装（代謝スループット、境界完全性、リソースレベル）
- [ ] NNコントローラーによる恒常性調整（ローカルLLM or 進化的NN）
- [ ] 感覚入力の実装（リソース勾配、他organism検出、環境条件）
- [ ] 環境摂動テスト（リソース枯渇、温度変化的イベント）
- [ ] 恒常性メトリクス: 摂動後の回復時間、内部状態の分散

**Go/No-Go (Week 4末)**:
- organismが摂動から回復するか？
- No → コントローラーを手動設計し、進化は後回し

### Week 5 (03/08-03/14): Phase 2 — 成長/発生 + 生殖

**目標**: ライフサイクルの完成

**成長/発生**:
- [ ] 最小シード → 成熟体への発生プログラム
- [ ] 代謝ネットワークの段階的拡張（成長に代謝コスト）
- [ ] swarmサイズの成長（新エージェントの追加/分化）

**生殖**:
- [ ] 代謝リソース蓄積による生殖条件
- [ ] ゲノムコピー + 変異 → 子organismの生成
- [ ] 子organismはシードから発生開始
- [ ] 生殖コスト（親のリソース分割）

### Week 6 (03/15-03/21): Phase 3 — 進化 + 統合

**目標**: 集団進化の動作確認

- [ ] 複数organism（10-50体）の同時シミュレーション
- [ ] 遺伝的変異（点変異、挿入/欠失、重複）
- [ ] 環境変動（リソース分布の変化、新たな摂動タイプ）
- [ ] 世代交代と自然選択の確認
- [ ] 適応度の世代推移トラッキング

**Go/No-Go (Week 6末)**:
- 7基準すべてが同時に機能しているか？
- No → 機能している基準に絞り、論文のスコープを調整（ピボットC）

### Week 7 (03/22-03/28): Phase 4 — 評価 + ビジュアライゼーション

**目標**: 定量評価 + 論文用figure

- [ ] 各基準のメトリクス計測（ランダムベースラインとの統計比較）
- [ ] 既存システムとの質的比較表の完成
- [ ] 非公式知覚評価の実施（同僚5-10名にデモを見せる）
- [ ] 論文用figure生成（シミュレーションスナップショット、メトリクス推移グラフ）
- [ ] 動画作成（補足資料用）

### Week 8 (03/29-04/01): Phase 5 — 論文執筆

**目標**: Full Paper (8p) 完成 + 投稿

- [ ] Abstract + Introduction（理論的基盤、C1対応）
- [ ] Related Work（既存システム比較、H2対応）
- [ ] System Design（ハイブリッドアーキテクチャの詳細）
- [ ] Experiments & Results（定量評価、C2対応）
- [ ] Discussion（限界、ピボット結果、将来研究）
- [ ] 参考文献の書誌情報完備（L1対応）
- [ ] ドメイン専門家によるレビュー
- [ ] **04/01 投稿**

**ピボット**: 間に合わない場合 → Extended Abstract (2-4p) に切り替え

## 技術アーキテクチャ概要

```
┌─────────────────────────────────────────────┐
│                 Environment                  │
│            (連続2D空間, Rust実装)              │
│                                              │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │Organism A│  │Organism B│  │Organism C│   │
│  │┌────────┐│  │          │  │          │   │
│  ││ Swarm  ││  │  ...     │  │  ...     │   │
│  ││Agents  ││  │          │  │          │   │
│  ││(10-50) ││  │          │  │          │   │
│  │├────────┤│  └──────────┘  └──────────┘   │
│  ││Metabol.││                                │
│  ││Network ││  リソース分布                    │
│  │├────────┤│  (光、栄養素等)                  │
│  ││Neural  ││                                │
│  ││Control ││                                │
│  │├────────┤│                                │
│  ││Genome  ││                                │
│  │└────────┘│                                │
│  └──────────┘                                │
└─────────────────────────────────────────────┘

各Organism内部:
- Swarm Agents: 境界維持、感覚、運動を担う個別エージェント群
- Metabolic Network: グラフベースの代謝経路（遺伝的にエンコード）
- Neural Controller: 恒常性調整 + 行動制御（進化的NN or ローカルLLM）
- Genome: 代謝ネットワーク + 発生プログラム + NNアーキテクチャを符号化
```

## 成功基準

### 最低限（Extended Abstract相当）
- 3基準以上が機能的に動作
- 単一organismの自律維持 >1000ステップ
- 代謝ネットワークの新規実装

### 目標（Full Paper相当）
- 7基準すべてが同時に機能
- 集団進化で適応が観察される
- ランダムベースラインとの統計的有意差
- 既存システムとの質的比較

### 理想
- 予期しない創発的現象の観察
- 非公式知覚評価で「生きている」判定
- Open-ended evolutionの兆候
